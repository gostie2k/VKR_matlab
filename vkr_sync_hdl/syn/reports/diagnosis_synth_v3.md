# Этап 3.12 — финальный протокол OOC-синтеза (v3)

**Дата:** 2026-05-20.
**Тулза:** Vivado 2023.2 (-mode batch).
**FPGA part:** xc7z020clg484-1 (Zynq-7020, CLG484, speed grade −1).
**Целевая частота PL-домена:** 100 МГц (период 10.000 нс), контроль на 150 МГц.

## 1. Резюме

Третья (финальная) итерация OOC-синтеза этапа 3.12 закрыта.
**Все шесть модулей замкнутой петли символьной синхронизации
закрылись по timing на 100 МГц с положительным WNS.**
V3-перепрогон (D7p) функциональную эквивалентность подтвердил:
все приёмочные критерии выполнены, MER @ EbN0=20 = 13.68 дБ
(деградация от baseline-D5 всего 0.20 дБ), фазовый jitter
вернулся к норме D6 после strobe-fix.

Архитектура (Gardner TED → PI → Mod-1 NCO → параболический Farrow,
sps_rx = 2, формат Q1.15, B_n·T = 0.02) и коэффициенты PI
(K1 = 0xFD8A, K2 = 0xFFEF) не изменены.

### 2.2. `rtl/sync_mod1_nco.v` (итерация 2)

Замена поведенческого деления:
```diff
- wire [.] cnt_shifted = {cnt_reg, {W_MU{1'b0}}};
- wire [.] mu_full     = (w_step != 0) ? cnt_shifted / w_step : 0;
- wire [W_MU-1:0] mu_calc = mu_full[W_MU-1:0];
+ wire [W_MU-1:0] mu_calc = cnt_reg[W_CNT-2 -: W_MU];
```
Аппроксимация `μ ≈ CNT_old · 2` через сдвиг (точна при `w_step = W_nom`).
136-уровневая CARRY-цепочка деления устранена. WNS: −47.27 → **+5.244 нс**.

### 2.4. `rtl/sync_farrow_parab.v` (итерация 3)

### 2.5. `rtl/sync_ted_gardner.v` (итерация 3)

## 5. Финальный критический путь sync_top @ 100 МГц

```
Slack (MET):       +1.720 нс
Source:            t_mul_mu_i_i_18/C      (внутри u_farrow,
                                            DSP P-register после pipeline)
Destination:       u_farrow/t_mul_mu_q__0/PCIN[0]
                                            (cascade-вход соседнего DSP48E1)
Data Path Delay:   6.796 нс
                      logic 4.895 нс (72.0 %)
                      route 1.901 нс (28.0 %)
Logic Levels:      2     (DSP48E1 = 1,  LUT2 = 1)
```

Путь теперь DSP→DSP через `PCIN` (cascade input соседнего DSP48E1)
с одним LUT для логики управления. Это естественный предел
конвейерного DSP-кластера ступени 3 Горнера на Zynq-7020 -1.
Дальнейшее улучшение требует разнесения двух соседних DSP
по разным частям колонки или явного manual placement —
не оправдано при текущем запасе +1.7 нс.

## 7. Запас по частоте (sync_top @ 150 МГц)

```
sync_top @ 150 МГц:  WNS = −0.802 нс
                     Critical path: тот же DSP→DSP через PCIN
                     Data path delay: 5.985 нс
                     Logic levels: 2 (DSP48E1=1, LUT2=1)
```

При периоде 6.667 нс реальная задержка пути = 6.667 + 0.802 = 7.469 нс,
**достижимая частота ≈ 1000 / 7.469 ≈ 133.9 МГц**. Это запас архитектуры
по тактовой частоте на 33.9 % сверх штатных 100 МГц AD9361 reference-проекта.

| Тактовая частота, МГц | Slack @ sync_top, нс | Статус         |
|-----------------------|---------------------:|----------------|
|  100 (штатная)        |             **+1.720**| ✓ closed       |
|  120 (≈ Fmax sync_top)|              ≈ +0.013| ✓ closed (граница) |
|  134 (теор. предел)   |             ≈ −0.001 | граница        |
|  150 (контроль)       |              −0.802  | ✗ не закрыт    |

Естественный потолок частоты определяется DSP-каскадом ступени 3
Горнера в Farrow. Дальнейшее повышение требует переработки
DSP-планировки (отдельные DSP-слайсы без PCIN) или
снижения разрядности коэффициентов — обе правки выходят за
рамки текущей архитектуры ВКР.

## 8. Особенности OOC-ограничений

Конфигурационные порты от AXI-Lite shim (`reg_k1`, `reg_k2`,
`reg_w_nom`, `reg_clamp`, `reg_agc_target`, `ctrl_*`)
объявлены `set_false_path` в соответствующих XDC:

| XDC                                            | False_path-порты                                    |
|------------------------------------------------|-----------------------------------------------------|
| `syn/constraints/sync_top_ooc.xdc`             | reset, ctrl_*, reg_k1/k2/w_nom/clamp/agc_target      |
| `syn/constraints/sync_top_ooc_150mhz.xdc`      | то же                                                |
| `syn/constraints/sync_loop_filter_pi_ooc.xdc`  | reset, k1, k2, clamp_lim                            |
| `syn/constraints/sync_agc_ooc.xdc`             | reset, agc_bypass, p_target                         |

В реальной интеграции (Block Design + AXI-Lite wrapper) эти
ограничения избыточны: регистры shim и регистры подмодулей
принадлежат одной clock-области, и timing между ними учитывается
стандартными правилами синхронного дизайна без дополнительных
ограничений.

## 9. Особенности инференса

### 9.1. DSP48E1 — все умножения инферированы как DSP

Подмодуль                | Умножения                              | DSP факт
-------------------------|-----------------------------------------|--------:
sync_loop_filter_pi      | e·K1, e·K2                              | 2
sync_ted_gardner         | mid_I·diff_I, mid_Q·diff_Q              | 2
sync_farrow_parab        | μ·c1, μ·t, etc. (3 ступени Горнера)     | 6

sync_mod1_nco            | (нет умножений)                         | 0

Округление `+ (1 << 14)` в PI поглощено DSP-pre-adder'ом. DSP-каскад
в Farrow (через `PCIN`/`PCOUT`) использован для оптимизации
маршрута между соседними DSP-слайсами.

### 9.2. sync_mod1_nco — деление полностью устранено

После итерации 2: 0 DSP48E1, 17 LUT, 29 FF. Критический путь —
7 уровней (5 CARRY4 вычитающего счётчика + 2 LUT underflow-detect).
Fmax ≈ 210 МГц.

### 9.3. sync_farrow_parab — 4 ступени pipeline сохранены

После итерации 3: 290 FF в OOC-режиме, 288 FF в интегрированном
sync_top. Retiming не разрушил архитектурную конвейерность
v_s1 → t_s2/mu_s2 → t_mul_mu_d/v0_d → xi_s3 (4 такта).

### 9.4. sync_loop_filter_pi — pipeline-ступень изолирована

После итерации 2: 70 FF (было ~52, добавились `vi_reg`,
`vp_s1_d`, `valid_s2` + retiming-копии); 194 LUT; 2 DSP48E1.

### 9.5. sync_ted_gardner и sync_agc — DSP-pipeline инферирован чисто

Все DSP-выходы зарегистрированы (P-register), внешние pipeline-регистры
не свёрнуты, дают предсказуемую +1 такт латентность.

## 10. Критические warning'ы

`syn/reports/critical_warnings.txt` содержит только:

1. Echo-строки `puts "ERROR: usage..."` из `run_ooc_synth.tcl` —
   ложные срабатывания парсера (это не реальный ERROR).
2. `CRITICAL WARNING [Vivado 12-4739] set_false_path: No valid
   object(s) found` для подмодулей `sync_farrow_parab`, `sync_mod1_nco`,
   `sync_ted_gardner`, синтезируемых с `sync_top_ooc.xdc`. У этих
   подмодулей нет AXI-Lite-портов, constraint становится no-op,
   синтез не страдает.

Реальных синтез-warning по RTL (`Synth 8-*` latches/sensitivity/
multi-driven) **не обнаружено**. DRC прошёл без замечаний.
