ПРОЕКТ

цифрового приёмо-передатчика на базе SDR».

Платформа:      Xilinx Zynq-7020 + AD9361
Инструмент:     Vivado 2023.2

==========================================================================
АРХИТЕКТУРА ПЕТЛИ СИМВОЛЬНОЙ СИНХРОНИЗАЦИИ
==========================================================================

Замкнутая петля:
    Gardner TED → PI-фильтр → Mod-1 NCO → кусочно-параболический Farrow (α=0,5)

Параметры:
    B_n·T   = 0,02
    ζ       = 1/√2
    K_p     = 2,7
    sps_rx  = 2
    Формат  = Q1.15

Коэффициенты PI (рассчитаны по Райсу):
    K1 = 0xFD8A   (≈ −0,0192)
    K2 = 0xFFEF   (≈ −5,13·10⁻⁴)

W_NOM = 0x8000 (1/sps_rx = 0,5)

==========================================================================
ДИАГНОСТИКА V3 (БЛОКЕР)
==========================================================================

Симуляция отрабатывает штатно: 1980 выходных символов = N_STIM/sps_rx
= 3960/2 точно. Но sync_out_symbols.txt показывает ОТСУТСТВИЕ
QPSK-созвездия:
    - распределение по 8 секторам фазы практически равномерное (12–14 %);
    - MER ≈ 12,7 дБ при ожидаемом ≥ 18 дБ;
    - RMS-амплитуда ≈ 20 351 при ожидаемой ≈ 11 585.

Стационарный режим (последние ~980 тактов из 3980):
    w_step:  mean=32772, std=108
    e_out:   |e|.mean=4458,  mean=−936
    v_pi:    mean=+4,  std=108
    vi_reg:  mean=−14, std=11
    mu_out:  mean=3175 (≈0,775), std=259  ← АНОМАЛИЯ

При W ≈ W_NOM значение μ должно быть около 0 или 1 (граница такта),
а не 0,775. Это означает, что Farrow интерполирует на ¾ пути между
двумя in_valid; TED получает y_mid в неверной позиции; знак e_out
непредсказуем; статической точки сходимости нет. Цепь замкнута и
реагирует, но не сходится.

Две рабочие гипотезы:
    А) Семантическое рассогласование mu_simple в sync_mod1_nco.v
       с временной разметкой TED (μ защёлкивается на «не том» CNT
       или имеется константный фазовый сдвиг).
    Б) Стимул gen_sync_top_stimulus.py (TIMING_OFF=0.3) стартует
       петлю вблизи anti-point Gardner TED.

Отделение гипотез выполняется контрольным прогоном с TIMING_OFF=0.0
и EBN0_DB=60.0.

==========================================================================
ИЗВЕСТНЫЕ ФАКТЫ — НЕ ПЕРЕПРОВЕРЯТЬ
==========================================================================

1. sync_mod1_nco.v использует mu_simple = cnt_reg[W_CNT-2 -: W_MU]
   (аппроксимация μ ≈ 2·CNT через сдвиг). Замена на полное деление
   не требуется — это штатное проектное решение.

2. В sync_top.v задержка строба — 2 такта (strobe_d1 → strobe_d2).
   Совокупно с регистрацией внутри NCO даёт 3 такта на выравнивание
   с Farrow. Код корректен.

3. В sync_top.v выражение w_step = reg_w_nom + v_pi_wire без явного
   $signed/$unsigned: поведение wrap-around mod 2^16, численно
   эквивалентно $unsigned($signed(reg_w_nom) + v_pi_wire).
   Код корректен.

4. Коэффициенты K1=0xFD8A, K2=0xFFEF в sync_params.vh корректны
   по знаку и значению.

5. В sync_loop_filter_pi.v строка 13: комментарий
   «K1 ≈ −0,0192 → 0xFB12» ошибочен (0xFB12 = −0,0385, правильно
   0xFD8A). Расхождение зафиксировано в дневнике ВКР как «известное
   несоответствие в комментарии модуля». ПРАВКИ КОДА НЕ ТРЕБУЕТСЯ.

==========================================================================
ФОРМАТ sync_internal.txt

9 столбцов, разделители — пробелы:
    1. clk_count   — номер такта
    2. nco_strobe  — мгновенный строб NCO
    3. strobe_d2   — задержанный строб (выравнивание с Farrow)
    4. cnt_reg     — текущее состояние счётчика NCO
    5. mu_out      — дробная фаза (Q0.W_MU)
    6. debug_w     — текущее w_step
    7. e_out       — выход TED
    8. v_pi_wire   — выход PI
    9. vi_reg      — состояние интегратора PI

В корне репо могут присутствовать:
    gen_8PSK.m, int_x2.m, int_x5.m, knk_pkrv_modem.v, parse_deb_data.m

Архитектура руководителя (knk_gardner_sync) принципиально отличается
от ВКР: sps=40, счётчик периода с целочисленным округлением (per_round)
и дробным остатком (dper, Q6.8), БЕЗ Farrow и БЕЗ Mod-1 NCO. Прямой
перенос невозможен. Однако коэффициенты K1=−2⁻², K2=−2⁻⁸ руководителя
и формула e=y_mid·(y_prev−y_curr) совпадают по знаку с реализацией
ВКР, что подтверждает корректность знаков K1=0xFD8A, K2=0xFFEF.
